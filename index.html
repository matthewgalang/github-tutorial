How to use Git on a Daily Basis:

  • git pull to sync with current repository
  • git branch branchname
  • git checkout branchname
  • Make edits and changes on local files
  • git status
  • git add filesthatwerechanged OR git add -A
  • git commit -m "description"
  • git push
  • git checkout masterbranch
  • Go to github to make pull request

Notes

If you don't commit with a description. You'll get a CLI error. To escape it, press esc key and enter ":wq"

NEVER MAKE A HABIT OF WORKING DIRECTLY ON THE MAIN BRANCH. ALWAYS MAKE A SEPARATE BRANCH AND NAME IT PROPERLY.

When in a branch, use "git pull" to update the branch to the branching node you're working on (if working on edits to master branch or working branch)

After pushing edits on a branch, go to github to submit pull request and wait for coding review.

If pull request is completed and reviewed, the request can be merged with the master branch or completed node.

Conflicts in code have to be worked on outside of code F2F. Whichever version of the code is agreed on can be submitted as a commit on the branch.
